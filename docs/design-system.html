Design Rules Summary

Don't
Use Solana purple/green as primary
Use cold gray backgrounds
Use Inter / Roboto / system fonts
Use thin 1px borders
Use thin 4px progress bars
Use flat buttons with no depth
Hardcode colors — always use tokens
Mix primary + secondary on same layer
Do
Teal primary, amber accent, Ink Teal secondary
Warm cream #FAFAF7 / dark #343431
Nunito display + Plus Jakarta Sans body
2.5px chunky borders everywhere
Fat 16px progress bars with inner glow
3D push buttons with bottom shadow
Use CSS vars → Tailwind tokens always
Primary = foreground action, Secondary = structure
Primary vs Secondary rule: Primary (#0D9488) = foreground action color (buttons, XP, progress, active states). Secondary (#0F2F2D) = background structure color (card accents, code panels, section dividers). They should never compete on the same visual layer. If two elements with these colors touch each other, something is wrong.
Dark mode rule: All colors adapt through CSS custom properties. Components use var(--token), never raw hex. Dark mode lifts primaries for readability and deepens shadows with rgba(0,0,0,0.35) instead of color-specific shadows. The push depth aesthetic is preserved via opacity-based shadows.
