Design Rules

Non-negotiable. Every rule here exists because a previous version broke it.

Never
Warm brown dark (#343431) or pure black (#0A0A0B)
Cold gray backgrounds
Solana gradient as primary brand color in UI
More than 3 button variants (Primary/Secondary/Accent)
Inter, Roboto, or system fonts
Level-up popup (removed in v8)
Level bar in leaderboard rows
Achievement badges without proof pill (if NFT minted)
Streak as circle days widget
Hardcode hex values in components
Always
Cool navy-slate dark (#0E1117 → #161B27 → #1C2333)
Warm cream light (#FAFAF7 → #FFFFFF)
Forest emerald as primary — our identity
Solana gradient ONLY on certificates + NFT elements
Nunito + Plus Jakarta Sans + JetBrains Mono
Stat strip + streak grid as one unified block
Wallet address in leaderboard, level badge + XP on right
Amber gradient ring on unlocked achievement badges
GitHub-style contribution grid for streak
CSS variables — always use tokens
The three popup rule: Only XP reward, Achievement Unlocked, and Certificate Minted. No level-up popup — level-up is communicated through the dashboard stat strip updating. Keeps reward moments focused.
Dashboard strip + streak: These two components must always be rendered together with no visual gap between them. The strip has no bottom border-radius, the streak block has no top border-radius. They share --card background. The result is one unified panel that communicates the learner's full status at a glance.
